Действуй как Senior System Architect и Fullstack-разработчик. Нам нужно разработать панель управления игровыми серверами и проектами (аналог Pterodactyl, но с кастомным интерфейсом).

1. Технологический стек:

    Backend (Core): Go (Golang) — для высокой производительности и многопоточности.

    Database: PostgreSQL (основные данные) + Redis (кэширование и сессии).

    Frontend: React (Vite) + TypeScript + Tailwind CSS. Использовать panel.css (предоставлю отдельно) как базовый стиль.

    Virtualization: Docker (обязательно). Реализовать демоны (Wings-like) на Go для каждой ноды.

2. Архитектура:

    Масштабируемость: Микросервисная архитектура. Один панель-мастер и множество удаленных нод (Remote Nodes), которые общаются через gRPC/Websockets.

    Безопасность: JWT авторизация, Role-Based Access Control (RBAC), SSH-ключи для доступа к контейнерам.

3. Функциональные требования:

    Управление нодами: Добавление нод через API, мониторинг состояния ресурсов (CPU, RAM, Disk) в реальном времени.

    Управление ресурсами: Создание контейнеров, лимитирование ресурсов (Cgroups), автозапуск.

    Файловый менеджер: Интегрированный FTP/SFTP (через встроенный сервер), редактор кода прямо в браузере.

    Консоль: Интерактивная консоль для каждого процесса через WebSockets (аналог tty).

    Создание проектов: Модульная система для быстрого деплоя (шаблоны для ботов, сайтов, игровых серверов).

4. Дизайн и UI/UX:

    Интерфейс должен быть максимально "чистым" и быстрым.

    Минимум лишних переходов — SPA (Single Page Application).

    В панели должны быть дашборды с графиками загрузки ресурсов.

Твоя задача:

    Предложить структуру проекта (папки backend, frontend, daemon).

    Спроектировать схему базы данных для хранения пользователей, серверов и нод.

    Описать протокол взаимодействия между мастер-панелью и демоном на ноде.

    Написать базовый код для старта (структура API в Go и React-компонент для списка серверов).»

Мои рекомендации по реализации:

    Почему Go для демона? Он идеально работает с Docker API и потребляет копейки оперативной памяти по сравнению с Node.js или PHP. Это даст тебе ту самую скорость, которую ты хочешь.

    Интеграция CSS: Когда начнешь разработку, держи panel.css в папке /src/styles/ фронтенда. При генерации компонентов проси нейросеть добавлять классы именно из твоего файла.

    FTP/SFTP: Не пытайся написать полноценный FTP-сервер с нуля. Используй готовые библиотеки для Go (например, goftp/server), чтобы не наделать уязвимостей.
