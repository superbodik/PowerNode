Для создания удобного процесса установки (по аналогии с Pterodactyl), тебе нужно создать автоматизированный Bash-скрипт установщика. Этот скрипт должен сам скачивать необходимые зависимости, настраивать Docker, создавать базы данных и конфигурировать систему.

Чтобы нейросеть написала качественный установщик для твоего проекта, отправь ей следующий промпт:
Промпт для генерации скрипта установки

«Действуй как DevOps-инженер. Нам нужно написать профессиональный bash-скрипт (install.sh) для автоматической установки моей панели управления серверами (аналог Pterodactyl).

Требования к скрипту:

    Диагностика: Скрипт должен проверять версию ОС (Ubuntu 24.04/22.04), права root и наличие критических зависимостей (curl, wget, git, tar).

    Интерактивность: Использовать удобное меню (аналог whiptail или dialog в терминале), чтобы пользователь выбирал:

        [1] Установить Мастер-панель (Backend + Frontend + DB).

        [2] Установить Демон-ноду (Wings + Docker).

        [3] Установить всё вместе.

        [4] Удаление текущей конфигурации.

    Автоматизация:

        Установка Docker и Docker Compose с официальных репозиториев.

        Установка и настройка MariaDB (автоматическая генерация пароля или запрос ввода).

        Клонирование репозитория проекта из GitHub.

        Создание системных сервисов (systemd) для панели и демона, чтобы они стартовали при загрузке ОС.

    Безопасность:

        Генерация случайных ключей API и секретов (APP_KEY, DB_PASSWORD).

        Настройка базового Firewall (UFW) для открытия только нужных портов (например, 80, 443, 2022 для SFTP).

    Логирование: Выводить красивые статусы процессов в терминал (например: [ OK ] Установка Docker завершена).

Твоя задача:
Напиши структуру скрипта install.sh. Используй переменные для версий, добавь функции для проверки установки каждого компонента. Скрипт должен быть модульным, чтобы я мог легко добавлять новые этапы установки в будущем.»
Почему это важно для твоей цели:

    User Experience: Если установка будет в стиле «нажал одну кнопку — получил работающий сервер», это сделает твою панель профессиональным продуктом.

    Использование dialog: В промпте я указал использование whiptail/dialog — это те самые «красивые» меню с выбором пунктов, которые ты видел при установке Pterodactyl. Нейросеть добавит это в код, и твоя консоль будет выглядеть как качественное ПО.

    Безопасность: Скрипт автоматически создаст все нужные пароли — тебе не придется прописывать их вручную в конфигах.

Небольшой совет по структуре:

Когда будешь развивать систему, раздели установщик на две части:

    install.sh: Основной скрипт для пользователя.

    scripts/: Папка внутри проекта, где лежат отдельные файлы для настройки сети, базы данных и Docker-контейнеров. Так код будет чище и его проще поддерживать.
